SettingsViewModel ToViewModel (IEnumerable< Setting > settings)
 Converts the Setting entities to a suitable SettingsViewModel that the presentation layer can use by performing various conversions. The algorithm for performing these conversions is as follows:

  1. At first, it attempts to perform a series of trivial conversions based on the Type property of the Setting entity. If no such conversion can be performed, an InvalidCastException is thrown. The following types are supported:
    • string
    • System.Boolean or bool
    • System.Int32 or int
    • System.Double or double
    Other types could be added, but are probably not needed.
  2. If the previous attempt failed, tries to invoke the constructor of the target type by using the CreateInstance of Activator and passing the Value property of the setting. In practice, this means that the algorithm attempts to invoke a constructor that takes a string as a parameter. If such a constructor doesn't exist, a MissingMethodException is raised.
  3. If the previous step failed, it tries to instantiate the Converter specified in the Setting by invoking its parameterless constructor. If that succeeds, it invokes its Convert method.
  4. If all fails, the conversion is considered impossible.

SettingsViewModel ChamiUI.BusinessLayer.Adapters.SettingsDataAdapter.ToViewModel ( IEnumerable< Setting >  settings)

Converts the Setting entities to a suitable SettingsViewModel that the presentation layer can use by performing various conversions. The algorithm for performing these conversions is as follows:

  1. At first, it attempts to perform a series of trivial conversions based on the Type property of the Setting entity. If no such conversion can be performed, an InvalidCastException is thrown. The following types are supported:
    • string
    • System.Boolean or bool
    • System.Int32 or int
    • System.Double or double
    Other types could be added, but are probably not needed.
  2. If the previous attempt failed, tries to invoke the constructor of the target type by using the CreateInstance of Activator and passing the Value property of the setting. In practice, this means that the algorithm attempts to invoke a constructor that takes a string as a parameter. If such a constructor doesn't exist, a MissingMethodException is raised.
  3. If the previous step failed, it tries to instantiate the Converter specified in the Setting by invoking its parameterless constructor. If that succeeds, it invokes its Convert method.
  4. If all fails, the conversion is considered impossible.
